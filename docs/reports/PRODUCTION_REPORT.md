## 5. Security Verification Report
- **Environment variables**: API keys are client-safe (Firebase standard practice).
- **Role permissions**: Enforced on the client-side (`ProtectedRoute`) and secured globally by backend rules (`database.rules.json` and `storage.rules`).
- **Unauthorized access**: Unauthorized users redirect to `/login`. Database rules block any direct REST access from unauthorized clients.
- **Protected routes**: `/dashboard` and nested paths are secured by `AuthContext`.
- **Session handling**: Managed entirely by Firebase Auth (JWT token state persistence).

## 6. Performance
- **Bundle size**: Handled natively by Vite. Modules use `React.lazy()` for code splitting on heavy admin routes.
- **Image optimization**: Utilizing webp where applicable, although dynamic content relies on users uploading appropriate formats.
- **Font loading**: Served securely, cached via Hosting rules.
- **Database requests**: Utilizes indexing (though rule evaluation keeps operations bound).

## 7. Deployment Instructions
1. Obtain Firebase configuration credentials for your actual project and populate them in your Hosting Platform's Environment Variables (or a `.env.production` file).
2. Install dependencies: `npm install`
3. Create production build: `npm run build`
4. Deploy to Firebase: `firebase deploy` (This handles rules, hosting, storage configuration based on `firebase.json`).

*Note: The platform provides its own hosting. If deploying outside AI Studio to a standalone Firebase project, follow the standard Firebase CLI initialization.*
