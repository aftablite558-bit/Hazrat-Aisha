# Contributing Guidelines

We welcome contributions to the Hazrat Aisha Academy project. Please follow these guidelines to ensure a smooth workflow.

## Development Workflow

1. **Branching:** Create a new branch for each feature or bugfix (e.g., `feature/student-module` or `bugfix/login-error`).
2. **Commits:** Write clear, concise commit messages.
3. **Pull Requests:** Submit a pull request against the `main` branch. Ensure all checks (linting, building) pass before requesting a review.

## Code Style

- Use TypeScript for all new code.
- Follow the established directory structure (`src/components`, `src/pages`, `src/services`, etc.).
- Use functional components and React hooks.
- Use Tailwind CSS for styling. Avoid inline styles.
- Ensure accessible markup (semantic HTML, ARIA roles, focus states).
